# Phase 1.1: Set Up Styling - Context

**Gathered:** 2026-02-05
**Status:** Ready for planning

<domain>
## Phase Boundary

Styling infrastructure for all future UI phases. This phase establishes the CSS framework, design tokens, base styles, dark mode, and a basic layout shell. No content or feature pages — just the foundation that Phases 2-8 build on.

</domain>

<decisions>
## Implementation Decisions

### CSS Framework
- **Bulma** via SCSS — not Tailwind, Bootstrap, or plain CSS
- **PurgeCSS** (`nuxt-purgecss`) to strip unused Bulma classes at build time
- SCSS preprocessor (`sass` package) — Vite handles it natively
- Bulma chosen for: SCSS-native, semantic classes, `.content` auto-styling for rendered markdown, lightweight with PurgeCSS

### Color Palette
- **Primary accent: `#b81a35`** — Type S red, sampled from the Type S image on acura.com
  - Alternate references: `#e82c2a` (acura.com CSS), `#E40521` (community approximation)
  - All three noted for Phase 8 review to confirm final brand color
  - Defined as a single SCSS variable (`$type-s-red` or similar) for easy future adjustment
- **Light mode:** Clean/minimal — white/light backgrounds, neutral tones, content-forward
- **Dark mode:** Warm neutral darks (~#171717 base, ~#262626 surfaces) — no blue undertone
  - Revisit dark mode palette in Phase 8 to confirm color/branding choices
- Acura red used **sparingly** — links, buttons, hover states, interactive elements only. Not in headers, nav bar, or hero backgrounds.

### Typography
- **Inter** (Google Font, self-hosted) — no external CDN requests
- System font stack as fallback
- Strong hierarchy: bold headlines, clear content flow (inspired by nuxt.com / content.nuxt.com)

### Icons
- **Lucide** — clean line icons, tree-shakeable, matches Nuxt reference site aesthetic

### Dark Mode
- Included in this phase (not deferred to Phase 8)
- **Trigger:** Defaults to system preference (`prefers-color-scheme`), user can override with manual toggle
- **Toggle location:** Nav bar (sun/moon icon)
- Preference stored in localStorage

### Link Styling
- Accent color text, no underline by default, underline on hover (Nuxt-style)

### Card & Surface Styling
- Subtle borders + rounded corners (matching Nuxt reference sites)
- Clean separation between content blocks

### Motion
- Subtle transitions set up in this phase — define transition variables (duration, easing) for hover states, dark mode toggle, page transitions

### SCSS Architecture
- **Partials structure:** `_variables.scss`, `_bulma-overrides.scss`, `_base.scss`, `_dark-mode.scss` → imported into `main.scss`
- **Override Bulma's variables** directly — set `$primary`, `$family-sans-serif`, etc. BEFORE importing Bulma
- **Comprehensive design tokens:** colors, spacing scale, border-radius, shadows, transitions — all as SCSS variables

### Component Styling Approach
- Bulma global classes for layout and base styling
- Component-specific styles in Vue SFC `<style lang="scss" scoped>` blocks
- Bulma's `.content` class configured to auto-style Nuxt Content rendered markdown (headings, lists, tables, links)
- Code blocks: basic styling only (monospace, background, padding) — no syntax highlighting

### Layout Shell
- Basic `default.vue` layout included in this phase: nav bar + main content area + footer skeleton
- Styled with placeholder content
- Hamburger menu on mobile (Bulma's built-in navbar collapse)
- Dark mode toggle in nav bar

### Responsive Strategy
- **Mobile-first** CSS (min-width media queries, Bulma's natural approach)
- **Bulma's default breakpoints:** mobile (<769px), tablet (769px+), desktop (1024px+), widescreen (1216px+), fullhd (1408px+)
- **Max content width:** Claude's discretion (balanced for content readability)
- **Responsive media defaults:** Global `max-width: 100%`, `height: auto` for images
- **Touch-friendly tap targets:** Minimum 44px for buttons/links on mobile

### Claude's Discretion
- Spacing/density: generous for landing pages, tighter for content-dense FAQ pages
- Max content width choice (likely ~1024px based on reference sites)
- Exact transition durations and easing curves
- Loading skeleton design if applicable
- Specific Bulma module imports (only import what's needed)

</decisions>
