## Decisions Made
- Used `@use` syntax exclusively (no `@import`) for Dart Sass 1.80+ compatibility with Vite 7
- Set `$scheme-h: 0` and `$scheme-s: 0%` to eliminate Bulma's default blue undertone from both light and dark modes
- Configured `dataValue: 'theme'` to bridge @nuxtjs/color-mode with Bulma's `[data-theme=dark]` selector
- Injected variables via vite `additionalData` as `vars` namespace so SFC style blocks use `vars.$token-name`
